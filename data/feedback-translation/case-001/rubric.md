# Evaluation Rubric: Case-001 (Hanna Lightweight Skirt)

## 1. 案例摘要
- **Input**: `Hanna Lightweight Skirt.pdf` (Sketched comments on design files).
- **Core Challenge**: 处理手写或电子标注中的非结构化意见，区分设计修改与商业承诺。

## 2. 评测指标 (Pass/Fail)

| 维度 | 关键检查点 (Must Have) | 失败判定 (Fail if...) |
| :--- | :--- | :--- |
| **翻译准确性** | 准确识别面辅料术语（如: shell fabric, lining, hem, piping）。 | 将面料缩写译错，或改变了修改意见的动作方向（如: increase 译为 decrease）。 |
| **风险标记** | 标注出所有涉及 `Delivery Date` 或 `Price` 的批注。 | 批注中包含日期或单价，但未在输出中打上 `[待确认]` 标签。 |
| **归并质量** | 将同一位置（如：领口）的多次修改意见聚合。 | 同一个修改点在列表中出现多次，且没有按部位或主题分组。 |
| **冲突检测** | 若批注 A 说“加宽 2cm”，批注 B 说“加宽 5cm”，必须列入冲突。 | 自动选择了其中一个值，或简单加和，未提示业务员存在冲突。 |

## 3. 评分标准 (1-5 分)

- **5分**: 完美。翻译地道，归并逻辑清晰，所有风险项均被标记，冲突 100% 检出。
- **4分**: 优秀。翻译准确，标记完整，但归并的分组命名略显生硬。
- **3分**: 合格。核心意见未遗漏，主要风险项已标记，但存在 1-2 处不影响大局的术语翻译瑕疵。
- **2分**: 不合格。遗漏了关键修改意见，或未能识别出明显的商务承诺（交期/价格）。
- **1分**: 严重错误。大面积翻译错误，或出现了严重的模型幻觉（捏造批注中不存在的要求）。
