# Role Memory: role-data-knowledge

## Purpose
- 定义外贸助手需要理解的数据对象、字段和知识边界。

## 关键数据对象
- `lead`: 客户与线索信息
- `inquiry`: 询盘正文、来源渠道、附件、时间
- `product_request`: 产品、规格、数量、目标价、认证
- `follow_up_task`: 下一步动作、截止时间、负责人
- `quote_readiness`: 是否具备报价条件及缺失项

## 验收结论/指标
- 工作流里出现的关键业务对象都要有稳定命名
- 字段抽取优先结构化，而不是只给自然语言摘要

## 风险与约束
- 不把缺失字段默认为真实值
- 不把外部知识库中的不确定信息当作事实缓存
