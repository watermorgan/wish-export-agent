# 验收基线（Export Agent）

## 1. 总体验收口径
- 所有能力必须服务于外贸销售场景，而不是泛聊天机器人。
- 每个工作流必须有清晰输入、工具调用边界、输出结构和人工接管点。
- 任何涉及客户资料、报价、订单和联系方式的数据处理都必须可审计。

## 2. MVP 能力基线
- 询盘识别：能区分 RFQ、售后、样品申请、价格咨询、渠道合作等主意图。
- 客户要素抽取：可结构化抽取国家、产品、数量、目标价、交期、认证要求。
- 跟进草拟：可生成中英双语回复草稿，并显式标注待人工确认项。
- 报价支持：可输出报价前检查清单，不直接伪造价格、库存或交期。
- 交接机制：当信息不足、风险过高或权限不够时，必须触发人工接管。

## 3. 工程基线
- 仓库必须保持独立 Git 边界。
- 本地记忆与技能清单必须可装配，且文件路径稳定。
- MCP 配置必须默认最小化，避免无关工具污染上下文。
- 秘钥、客户隐私数据和生产凭据不得进入版本库。

## 4. 安全与合规基线
- 不输出未经确认的贸易承诺，包括最终价格、交付时效、认证结论和物流承诺。
- 不向未授权外部系统发送客户数据。
- 对客户联系人、邮箱、电话、地址等敏感信息，默认按最小必要原则处理。

## 5. 当前初始化完成标准
- 已创建本地 `memory/`、`skills/`、`.codex-bridge.json` 和 `.mcp.json`
- 当前目录已成为独立 Git 仓库
- 已有首个可提交的初始化基线
