# Gemma4 B-only 差异说明（2026-04-07）

## 对比口径

本说明只比较同一批已成功的 A/segment 结果下，不同 B 模型带来的差异。

快速阅读版可见：

- `docs/project/gemma4-b-only-review-package.md`

- 基线：当前阶段版 PDF / snapshot
  - 目录：`.tmp/business-review-pdfs/`
- 本地对比版：仅把 B 重译为 `gemma-4-31B-it-Q3_K_M.gguf`
  - 最新目录：`.tmp/gemma4-b-only-review-refined-v4/`
- 关键约束：
  - 不重跑 A
  - 不改变 segment 集合
  - 不改变 renderer 逻辑
  - 因此这里看到的差异，主要反映 B 模型的翻译风格与术语偏好

## 总体结论

本地 `gemma4` 作为 B 时，最大的差异不是“完全翻不出来”，而是：

1. **更直译**
   - 更倾向保留英文原结构或原单位
   - 较少主动压缩成业务人工稿短句

2. **更容易翻元信息**
   - 对 `HIVER 26 / DOSSIER STYLE / DESIGNER / GRAPHISTE / ACHETEUR / copyright` 等页眉/管理字段更容易给出中文
   - 这不是主链缺块，而是风格和 suppress 口径差异更大

3. **业务标签表达更“机械”**
   - 颜色：`02#黑色` 会变成 `02 黑色`
   - 口袋/拉链/面料类短句更接近字面直译
   - 人工稿式压缩表达较少

4. **在固定 A/segment 的前提下，4 个代表样本都能形成完整业务预览**
   - `M422123 / M441083 / M445033 / M415013` 全部 `businessPreviewReady=true`
   - 说明本地 `gemma4` 作为 B 的“承接能力”已成立
5. **refined 版已保留阶段版 suppress 结果**
   - 不会再把阶段版本来留空的 footnote/meta 条目重新翻出来
   - 当前 4 个样本里，本地 B-only 与阶段版的“有效已翻条数”已经完全对齐：
     - `M422123`: `19 / 24`
     - `M441083`: `38 / 46`
     - `M445033`: `36 / 60`
     - `M415013`: `30 / 48`

## 分样本差异

以下差异基于 `.tmp/gemma4-b-only-review-refined-v4/`，已经排除了“阶段版原本 suppress 的条目被重新翻出”这一类假差异。

### M422123

主要差异：

- 元信息更容易被翻出：
  - `HIVER 26 (26H)`：阶段版 `款号：HIVER 26 (26H)`，本地版 `冬 26 (26H)`
  - `DOSSIER STYLE`：阶段版 suppress，本地版 `款式档案`
- 业务术语更直译：
  - `02 NOIR`：阶段版 `02#黑色`，本地版 `02 黑色`
  - `POCKETING...`：阶段版 `袋布：涤棉磨毛斜纹，配色同面布`，本地版 `口袋布：刷毛涤棉斜纹布，颜色：与面料同色`
  - `AUTOBLOCK ZIPPER...`：阶段版 `枪色自动锁头闭尾拉链`，本地版 `自动锁止拉链 - 枪色，前中门襟`
  - `Back elasticated waistband`：阶段版 `后腰部橡筋`，本地版 `后腰弹力腰带`

判断：

- 业务信息完整性没有问题
- 差异主要在“是否像人工稿短句”
- refined 后，页眉/管理字段不会再额外冒出来，差异已经更集中在术语和句式
- 当前剩余差异已收缩到 `5` 条，主要是状态行和面料规格压缩方式

### M441083

主要差异：

- 面料/填充/里布类更偏说明式直译：
  - `SHELL FABRIC...`：阶段版更短、更像工艺单；本地版保留更多原句结构
  - `PADDING...`：阶段版 `60g大身，40g袖子`，本地版 `填充物：身前60GRS，袖子40GRS，SORONA品质`
  - `BODY LINING...`：阶段版 `身里春亚纺 黑色`，本地版 `身片里布：哑光尼龙里布 颜色：02 NOIR`
- 工艺说明更字面：
  - `MIDDLE BACK printed reflective line...`：阶段版 `后背反光面胶`，本地版更接近逐词翻译
  - `Top back yoke THERMOFUSED FLAP`：阶段版 `后浮水压双面胶`，本地版 `后背上过肩：热熔贴片`

判断：

- 本地版可读，可用来对照
- refined 后条数与阶段版一致，说明当前差异主要是术语风格，不是 suppress 口径失控
- 在反光、压胶、工艺缩写这类服装术语上，当前阶段版更贴近人工稿
- 当前剩余差异已压到 `2` 条，主要是：
  - `压胶` vs `复合`
  - `所有工艺需防水` vs `所有做工需防水`

### M445033

主要差异：

- 这份样本的差异最明显，集中在：
  - 面料/里布/填充表达
  - 拉链/口袋/新主标的短句压缩
  - 页眉元信息翻译
- 典型例子：
  - `48 MARINE`：阶段版 `48#海军蓝`，本地版 `48 海蓝色`
  - `SHELL FABRIC...`：阶段版 `面料1 与M245013相同面料`，本地版保留完整规格说明
  - `PADDING...`：阶段版 `填充：与M145023相同`，本地版 `填充物：重量及品质同M145023，100%再生聚酯纤维`
  - `NEW LOGO LABEL`：阶段版 `新主标`，本地版 `新LOGO标`
  - `INVISIBLE ZIPPER...`：阶段版 `3#隐形拉链侧`，本地版 `隐形拉链：前口袋，颜色与面料一致`

判断：

- 本地版业务块齐全，但更像“详细说明稿”
- 当前阶段版更像人工工艺单写法
- refined 后这份样本最典型的差异就是“术语压缩不足”，不是漏翻
- 当前剩余差异已收缩到 `4` 条，属于可继续打磨但不影响整体验证结论的范围

### M415013

主要差异：

- 术语和标签的人工稿贴合度仍然差一些：
  - `PROTO #1`：阶段版 `OP1`，本地版 `首样 #1`
  - `MATCHING COLOR WITH OUTSHELL FABRIC`：阶段版 `顺色`，本地版 `颜色：同面料`
  - `AS YOUR ATTACHMENT SAMPLE`：阶段版 `尺寸和版型同参考样衣`，本地版 `参照附件样衣`
  - `CUFF + BELT + COLLAR FABRIC`：阶段版 `袖口、底摆、领材料`，本地版 `袖口+腰带+领子面料`
  - `SAME COTTON FACE LOOK BUT SHAVED POLAR FLEECE TO BE THINNER`：阶段版 `比主身摇粒绒面料更薄一些`，本地版更接近解释性直译
- 元信息和人员名也更容易被翻出

判断：

- 本地版已经能承接这份样本
- refined 后，`M415013` 的业务条数已经与阶段版一致，说明主问题不再是“能不能翻出来”
- 但在 `OP1/OP2`、`顺色`、`参考样衣` 这类业务黑话上，阶段版明显更成熟
- 当前剩余差异已压到 `1` 条，只剩 `11 Ecr` 这种局部标签解释争议

## 当前建议

如果目标是：

- **比较本地 gemma4 的翻译风格**
  - 这组 B-only 结果已经够用了

- **判断能否替代当前阶段版**
  - 当前还不能
  - 原因不是“漏翻”，而是：
    - 元信息 suppress 不够稳
    - 业务术语更直译
    - 人工稿式短句压缩不足

## 最值得继续优化的方向

不要再优先折腾本地 A。

对本地 `gemma4` 作为 B，后续最值的是：

1. 收术语归一
   - `OP1 / OP2`
   - `顺色`
   - `新主标`
   - `海军蓝`
   - `后腰部橡筋 / 斜插侧袋 / 单开线口袋` 这类服装工艺短语

2. 收“详细说明”向“人工稿短句”压缩
   - 尤其是面料、里布、填充、拉链、工艺说明

3. 只在必要时补少量局部标签解释
   - 例如 `11 Ecr` 这类孤立局部标签，是否按颜色解释还是按裁片解释，需要结合业务口径再定，不适合再盲目自动归一
