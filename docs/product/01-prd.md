# 外贸助手 V1 产品需求文档

## 1. 文档目标
定义外贸助手 V1 的产品目标、目标用户、核心问题、范围边界和首批业务链路，为后续设计与开发提供统一口径。

## 2. 产品定位
外贸助手是一个面向企业内部私有部署场景的 AI 工作台，服务不懂技术的一线外贸团队。产品不采用开放式自由聊天范式，而是围绕可配置技能卡片和人工触发组合，帮助用户完成高频、重复、需要审校的外贸办公任务。

参考方向：
- 形态上参考统一工作台模式：统一工作区、统一资料入口、统一执行区
- 能力组织上参考 CoPaw 的 `workspace + skill catalog + composition` 思路

本产品不会照搬开放式多 Agent 产品边界，不以“自由探索式 Agent”作为 V1 主体。

## 3. 目标用户
### 3.1 业务员
- 负责接收客户询盘、处理工艺单、整理资料、做初步回复
- 需要上传文件、调用技能、查看结构化结果、修改与导出

### 3.2 主管
- 负责审核输出、维护技能模板、定义组合规则、复盘结果质量
- 需要查看执行记录、发布技能、设置审核规则和复用模板

## 4. 核心问题
- 一线业务员需要反复处理工艺单、批注、邮件和客户上下文，任务重且标准化程度高
- 现有 AI 聊天方式输出不可控，容易给出未经确认的承诺
- 团队需要把经验沉淀为可复用技能，而不是每个人重复写提示词
- 处理链路中存在明确人工确认节点，不能被“自动 Agent”直接跳过

## 5. 产品目标
- 提供一个安全可控的内部工作台，让业务员能以“上传资料 + 选择技能 + 审校结果”的方式完成外贸日常任务
- 将高频任务沉淀为可配置技能，并支持人工触发的技能组合
- 所有可能形成对外承诺的内容必须显式标记为“待确认”
- 为后续接入 Feishu 等 bot 渠道保留统一服务接口，但不把 bot 作为 V1 主入口

## 6. 非目标
- 不做开放式泛聊天机器人
- 不做自动报价与自动对外发送
- 不做自动写入 ERP、CRM、邮件系统
- 不做无人工确认的全自动连续执行
- 不按公网 SaaS 或多租户架构设计 V1

## 7. V1 范围
### 7.1 主入口
- 内部 Web 工作台

### 7.2 首批技能包
- 工艺单/BOM 整理
- 多意见翻译与归并
- 客户回复草拟

### 7.3 核心能力
- 上传外贸常见文件
- 选择单技能执行
- 选择技能链执行
- 生成结构化输出
- 标记待确认项
- 审核、导出、留痕

## 8. 成功标准
- 业务员能在一个工作台里完成首批 3 类任务，不依赖自由聊天式交互
- 主管能审核结果，并能配置技能与组合规则
- 所有结果都有执行记录、来源材料和人工确认状态
- 对外承诺类内容被显式阻断或标记，不允许静默输出

## 9. 核心业务链路
### 9.1 工艺单 -> 面辅料 BOM
#### 目标
帮助业务员把工艺单、面料说明、辅料要求等资料整理成结构化 BOM。

#### 输入
- 工艺单文档
- 面料/辅料说明
- 相关图片或批注资料
- 用户任务说明

#### 输出
- 结构化面辅料 BOM 表
- 缺失字段清单
- 待人工确认项
- 可导出结果

#### 人工确认点
- 料号、规格、颜色、单位、损耗等关键字段
- 不确定项、识别冲突项和图文不一致项

#### 失败回退
- 无法稳定识别字段时，输出“需人工补录字段清单”
- 文件解析失败时，提示重新上传或拆分文件

#### 安全边界
- 不伪造未在材料中出现的 BOM 字段
- 不自动生成采购或成本结论

### 9.2 多意见/批注 -> 翻译与归并
#### 目标
帮助业务员快速处理大量意见、批注和跨语言沟通内容。

#### 输入
- 中文/英文意见文本
- 批注导出内容
- 多份附件或聊天记录
- 用户任务说明

#### 输出
- 双语翻译结果
- 去重后的意见列表
- 归并后的主题分组
- 待确认术语或歧义项

#### 人工确认点
- 专业术语翻译
- 模糊表达、否定表达、责任归属
- 最终对客户可发送版本

#### 失败回退
- 语境不足时，标记“需补充上下文”
- 多文档冲突时，分组展示并要求人工选择

#### 安全边界
- 不把模糊意见翻译成确定承诺
- 不省略影响责任归属的关键句

### 9.3 客户资料/上下文 -> 英文回复草稿
#### 目标
基于客户资料、历史上下文和附件，为业务员输出可审校的英文回复草稿。

#### 输入
- 客户邮件或聊天记录
- 相关附件
- 业务员补充说明
- 可选的回复风格或目标

#### 输出
- 英文回复草稿
- 中文备注
- 缺失信息清单
- 待确认承诺项

#### 人工确认点
- 价格
- 交期
- 认证与合规结论
- 物流与付款承诺

#### 失败回退
- 上下文不足时，生成“澄清问题清单”
- 附件无法解析时，保留文本链路并提示资料缺失

#### 安全边界
- 不直接输出未经确认的商业承诺
- 对高风险措辞必须标注“待确认”

## 10. 功能需求
### 10.1 工作台
- 支持上传多文件和输入任务说明
- 展示技能目录、推荐组合、执行结果和审核状态

### 10.2 技能执行
- 支持业务员手动选择技能
- 支持按模板组合技能
- 支持保留中间产物，不允许黑盒执行

### 10.3 审核与导出
- 支持主管审核通过/退回
- 支持查看待确认项、来源引用和人工修改版本
- 支持导出结构化结果或草稿

### 10.4 留痕与审计
- 保存执行请求、上传文件、技能链路、中间结果和最终结果
- 保存人工编辑与审核记录

## 11. 第二阶段预留
- 自动报价
- 自动发送外部消息
- 自动写入 ERP/CRM
- 无人工确认的自主串行执行
