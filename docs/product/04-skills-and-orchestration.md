# 外贸助手 V1 技能与编排文档

## 1. 文档目标
定义 V1 技能目录、技能卡片结构、组合规则和人工触发编排原则，避免后续开发退化成自由提示词堆叠。

## 2. 编排原则
- 技能是最小业务能力单元
- 技能组合必须由用户明确选择或由模板推荐后人工确认
- 系统可以推荐组合，但不能在 V1 中无确认地自主串行执行
- 高风险步骤必须插入人工确认节点

## 3. 技能卡片定义
每张技能卡片都必须对应一个 `SkillDefinition`，至少包含以下字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 技能标识 |
| `name` | 技能名称 |
| `purpose` | 用途说明 |
| `inputRequirements` | 输入要求 |
| `outputSchema` | 输出结构 |
| `reviewCheckpoints` | 人工确认点 |
| `composableWith` | 可组合技能 |
| `riskLevel` | 风险等级 |

## 4. V1 技能目录
### 4.1 BOM 整理技能
- 用途：把工艺单和相关资料整理成结构化 BOM
- 输入：工艺单、说明资料、补充备注
- 输出：BOM 表、缺失字段、冲突字段、待确认项
- 风险等级：中
- 关键确认点：规格、单位、颜色、损耗、未识别字段

### 4.2 意见翻译技能
- 用途：将多来源意见进行双语翻译
- 输入：批注、意见文本、聊天记录
- 输出：逐条翻译结果、术语清单、歧义提示
- 风险等级：中
- 关键确认点：专业术语、否定表达、责任归属

#### PDF 模式补充
- 当输入是 PDF 批注/工艺页时，意见翻译技能不应只输出文本对照，而应进一步输出：
  - 正式 annotated PDF / 预览入口
  - 结构化人工复核建议（HITL）
  - 稳定 skill 协议 `pdf_translation_skill_v1`
- 这意味着 PDF 模式下的“意见翻译技能”已经不只是 prompt，而是复用同一套 `pdf-pipeline + snapshot + render + humanReviewGuide` 结果层。

### 4.3 意见归并技能
- 用途：对翻译后的意见去重、归并和主题整理
- 输入：原文意见或翻译结果
- 输出：主题分组、归并列表、冲突项
- 风险等级：低
- 关键确认点：冲突意见是否合并、主次优先级

### 4.4 客户回复草拟技能
- 用途：基于上下文生成可审校英文回复
- 输入：客户上下文、附件、任务目标
- 输出：英文回复草稿、中文备注、待确认项
- 风险等级：高
- 关键确认点：价格、交期、认证、付款、物流

## 5. 技能组合规则
### 5.1 单技能执行
- 适用于单一明确任务
- 例如只做 BOM 整理，或只做客户回复草稿

### 5.2 技能链执行
- 适用于需要连续产出的任务
- 由用户手动选择多个技能，或选择预定义模板

### 5.3 主管配置/发布技能
- 主管可定义技能说明、输入要求、风险等级和组合关系
- 主管可发布技能模板给业务员

### 5.4 业务员调用技能
- 业务员只调用已发布技能或模板
- 业务员不能绕过必填输入和人工确认规则

## 6. V1 推荐技能链
### 6.1 翻译与归并链
- `意见翻译技能` -> `意见归并技能`

#### 适用场景
- 多份批注
- 多语言沟通意见
- 大量文本需要整理

#### 阻断条件
- 原文缺失
- 文本质量过低
- 关键上下文不足

### 6.2 回复准备链
- `意见翻译技能` -> `客户回复草拟技能`

#### 适用场景
- 先理解多来源意见，再生成对外回复

#### 阻断条件
- 价格或交期信息未确认
- 责任归属不明确

### 6.3 工艺整理链
- `BOM 整理技能`

#### 适用场景
- 工艺单和资料已齐全，目标是形成 BOM

#### 阻断条件
- 工艺单解析失败
- 关键字段缺失过多

## 7. 禁止自动跳过的人工确认点
- 所有价格类表述
- 所有交期类表述
- 所有认证和合规类表述
- 所有付款与物流类表述
- 所有字段冲突与识别不确定项
- 所有歧义翻译与责任归属判断

## 8. 模板发布规则
- 模板只能由主管发布
- 模板必须绑定：
  - 适用场景
  - 输入要求
  - 步骤顺序
  - 阻断条件
  - 人工确认点
- 模板必须具备清晰完成产物，不能只有自由文本描述

## 9. 第二阶段预留
- 自动推荐最优技能链
- 按上下文自动补齐技能链
- 自动对外发送消息
- 自动写入 ERP/CRM
- 自主串行执行且仅做抽样审核
